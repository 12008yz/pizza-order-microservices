План выполнения проекта

ЭТАП 1: Backend — доработка микросервисов для MVP (готово 4 из 9) (1 неделя)

Сервис заказов:

Сервис покрытия - автодополнение адресов:
 API автодополнения адресов беру из базы данных покрытия
 Возвращает подсказки адресов из нашей базы покрытия (не Яндекс Геокодер)
 В конце заменится на реальную базу покрытия

Микросервисы проекта
Всего: 18 микросервисов

Обязательные для MVP (9):
API Gateway Service — маршрутизация запросов
Auth Service — аутентификация пользователей (готов)
User Service — профили пользователей (готов)
Location Service — управление адресами (готов)
Availability Service — проверка доступности
Provider Service — провайдеры и тарифы (есть ошибки в тестах)
Equipment Service — оборудование провайдеров (готов)
Order Service — обработка заявок
Notification Service — отправка уведомлений

Рекомендуемые (4):
Scheduling Service — планирование визитов
Consultation Service — консультации клиентов
Integration Service — интеграция с API
File Service — управление файлами

Для CRM панели (3):
CRM Service — управление заявками
Workflow Service — рабочие процессы
Admin Service — администрирование системы

Опциональные (2):
Search Service — поиск по данным
Analytics Service — аналитика и отчеты


ЭТАП 2: Frontend пользовательского приложения (2-3 недели)
Фрейм 1: Главная и поиск
- Поиск адреса с автодополнением из базы данных покрытия
- Пользователь вводит адрес, получает подсказки из базы покрытия
Фрейм 2: Результаты поиска (моковые данные)
- Список провайдеров по выбранному адресу
- Проверка доступности через Микросервис покрытия
Фрейм 3: Форма заявки
- Многошаговая форма (выбор тарифа, личные данные)
- Выбор оборудования (Микросервис оборудования)
- Создание заявки (Микросервис заявки подключения)
Фрейм 4: Статус заявки
- Отображение статуса заявки
- История изменений
